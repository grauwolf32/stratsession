## Notable findings that the README-driven reports missed

Going from README-level analysis to code-level surfaced a few corrections / new observations worth highlighting separately:

1. **Allama's "WebAssembly Python sandbox" claim is wrong.** The implementation is **nsjail** (Linux user/mount/net/pid namespaces) with `cap_add: SYS_ADMIN`, plus an AST-allowlist fallback (`SAFE_STDLIB_MODULES` in `allama/sandbox/safe_executor.py`). Repo-wide grep for `wasm|webassembly|pyodide|rustpython|wasi` returns zero matches. See [`allama.md`](./allama.md) §"Sandboxing".
2. **Allama's "80+ integrations" resolves to ~20 first-class Python adapters + 349 YAML "template tool" files across 49 vendor directories.** The number is real if you count templates; the architecture per-tool varies. See [`allama.md`](./allama.md) §"80+ integrations".
3. **Brutus ships a 2 MB Rust IronRDP module compiled to WASM and drives it from Go via wazero with 7 host functions.** This is the most interesting engineering choice in the inventory — lets a single-binary, zero-CGO Go program speak NLA/CredSSP correctly without rewriting RDP. The sticky-keys/utilman backdoor detector is layered on top with a deterministic pixel-diff heuristic + optional Claude Vision second opinion that fuses verdicts with explicit confidence arithmetic. See [`brutus.md`](./brutus.md) §"RDP via IronRDP-in-WASM".
4. **Plumber's "positive-evidence" degraded mode** is genuinely novel: `GitHubMetadataClient` returns zero-valued metadata on auth failure (`collector/github_metadata.go:154-176`), and every metadata-dependent Rego policy is written to fire only on positive evidence — so a missing token quietly silences API-dependent rules instead of failing the run. See [`plumber.md`](./plumber.md) §"Soft-degrade mode".
5. **mquire's kallsyms scanner bootstraps `kallsyms_token_index` by *constructing* the expected u16 array from the recovered token bytes** and pattern-matching it back into memory (`kallsyms.rs:814-841`). It also requires kernel ≥ 6.4 (commit `404bad70fcf7` changed the output order). Cross-source rootkit-detection dedup is deliberately punted to SQL — `source='task_list'` vs `source='pid_ns'` columns + `FULL OUTER JOIN`. See [`mquire.md`](./mquire.md) §"Kallsyms scanner" and §"Multi-source task discovery".
6. **Little Snitch for Linux's `GPL-2.0` license is an ABI requirement, not just a license preference.** The eBPF `static LICENSE: [u8; 4] = *b"GPL\0";` declaration at `ebpf/src/main.rs:374-376` is checked by the kernel's `BPF_PROG_LICENSE` gate — helpers like `bpf_get_socket_cookie`, `bpf_probe_read_kernel`, and `fentry`/`fexit` tracing only work for GPL-compatible programs. Distributing under any non-GPL-compatible license would result in kernel rejecting program load. See [`littlesnitch-linux.md`](./littlesnitch-linux.md) §"License + ABI implications".
7. **CERT UEFI Parser ships a custom mypy plugin** (`mypy.py`, registered in `mypy.ini`) that bridges declarative Construct field definitions into static types so the ~25k-LOC dynamically-typed parser can ship under `strict = True`. Uncommon engineering for a firmware tool. See [`cert-uefi-parser.md`](./cert-uefi-parser.md) §"Type stubs".
8. **cloud-audit's What-If simulator** specifically takes a `relationships=None` mode in `correlate.detect_attack_chains` so it can re-run on a saved report without any AWS access — the trade-off is that it cannot evaluate the 7 relationship-dependent chain rules (AC-01, AC-02, AC-05, AC-07, AC-23, AC-24, AC-29). The trade-off is documented in code, not hidden. See [`cloud-audit.md`](./cloud-audit.md) §"What-If simulator".
9. **Betterleaks's `failsTokenEfficiency()` CEL primitive uses an offline-embedded `cl100k_base` BPE tokenizer** (`detect/tiktokenloader.go:13` embeds `assets/cl100k_base.tiktoken.gz`). Chars-per-token ratio is the rare-string signal (prose tokenizes efficiently; random secrets do not). The validator pool has a sha256-keyed + singleflight result cache so a monorepo with the same leaked key in 500 files hits the upstream API once. See [`betterleaks.md`](./betterleaks.md) §"BPE naturalness scorer".

### Round 2 findings

10. **⚠ Caido is closed source.** The public `caido/caido` GitHub repo — mirrored at `sources/perimeter/caido/` — contains only brand assets, issue templates, a triage-bot prompt, and a security policy. No Rust workspace, no `Cargo.toml`, no proxy core. The brief's `reports/perimeter-stack.md` framing of Caido as a "Rust Burp-alternative" is accurate as a product description but **the source is not on GitHub**. The one source-grounded architectural fact: `SECURITY.md` explicitly states plugins have **unrestricted host access** — Caido is *not* Wasm-sandboxed. See [`caido.md`](./caido.md).
11. **Trivy collapsed every misconfig source onto one Rego engine.** Dockerfile reconstructed from image history, Terraform on disk, in-cluster K8s YAML, and Helm templates all funnel through `pkg/iac/rego/scanner.go`. Most competitors bolt a separate engine per IaC type or shell out to `tfsec`/`checkov`/`kube-bench`. See [`trivy.md`](./trivy.md) §"`pkg/iac/`".
12. **Buttercup uses LLMs as fuzzer peers, not assistants.** `VulnBaseTask` pushes successfully-crashing LLM-generated inputs onto the same `QueueNames.CRASH` stream the `FuzzerBot` writes to — downstream consumers (tracer, deduper, patcher) cannot tell whether a finding came from libFuzzer or from Claude. See [`buttercup.md`](./buttercup.md) §10.
13. **Nuclei does not implement its own DSL.** It's a 141-line shim over `projectdiscovery/dsl` which wraps `Knetic/govaluate` — only `resolve()` and `getNetworkPort()` are Nuclei-specific functions. The same evaluator runs every matcher/extractor/template-condition across HTTP/DNS/SSL/headless/etc. See [`nuclei.md`](./nuclei.md) §6.
14. **LibAFL ships libFuzzer drop-in as a two-crate split with `include_bytes!`-embedded prebuilt runtime.** The runtime crate is *excluded* from the workspace at `Cargo.toml:55` and built separately so the user's `-fsanitize-coverage` flags can't accidentally instrument the fuzzer itself. See [`libafl.md`](./libafl.md) §7.
15. **Hadrian's planner uses a closed-vocabulary LLM with hard validator.** Any planner step that references a nonexistent template/role/operation is silently dropped *before* a request is sent (`pkg/planner/planner.go:94`) — the LLM cannot widen the attack surface beyond what Hadrian's own template library supports. See [`hadrian.md`](./hadrian.md) §8.
16. **Seclab Taskflow Agent's YAML grammar loads from any installed Python package.** `importlib.resources.files(package).joinpath(filename+".yaml")` (`available_tools.py:117-124`) turns the dotted-module-path into a plugin source — toolboxes/personalities/taskflows can live in any pip-installed package. See [`seclab-taskflow-agent.md`](./seclab-taskflow-agent.md) §3.
17. **PatchLeaks bootstraps its language-builtin lists by exec-ing user-installed runtimes.** `php -r 'echo json_encode(get_defined_functions()...)'`, `python3 -c 'print(json.dumps(dir(builtins)))'`, `ruby -e 'puts (Kernel.methods + ...)'` are invoked at startup. Unusual — most static analysers ship frozen lists. The Go path is a stub: `exec.Command("go", "doc", "builtin")` is called but the output is discarded. See [`patchleaks.md`](./patchleaks.md) §4.

### Round 3 findings

18. **AFL++ "binary-only" modes are submodule fetches at pinned commits, not in-tree forks.** `qemu_mode/`, `frida_mode/`, `unicorn_mode/`, `nyx_mode/`, `coresight_mode/` each have a `build_*.sh` that clones an external `qemuafl`/`unicornafl`/`libnyx` repo. Evaluating AFL++ means evaluating a *family* of repos; a build that succeeds today may fail if a submodule's upstream rewrites tags. See [`aflplusplus.md`](./aflplusplus.md) §7.
19. **DefectDojo registers scanners by directory convention, not enumeration.** Drop `dojo/tools/<name>/parser.py` with a class named `<Name>Parser` exposing `get_scan_types()` — no manifest, no `__init__.py` edits, no migration. The discovery loop (`dojo/tools/factory.py:108-128`) builds the dispatch table at import time, and `Test_Type` rows are conjured on first parser use. This is what sustains the 221-scanner long tail. See [`defectdojo.md`](./defectdojo.md) §3.
20. **DefectDojo's multi-tenancy is at the query-helper boundary, not the DB.** No row-level security, no separate connection per tenant. Every leak risk is in whether the ViewSet remembered to call `get_authorized_<model>(permission)`. See [`defectdojo.md`](./defectdojo.md) §9.
21. **Coraza has compile-time regex prefiltering + tinygo build targets.** `internal/operators/rxprefilter.go` parses every `@rx` pattern with `regexp/syntax` to extract literal sets and detect pure `^literal$` for an NFA bypass. 51 files carry `//go:build tinygo` or `//go:build !tinygo` tags so the same engine compiles as both a Go middleware and a proxy-wasm Envoy filter. See [`coraza.md`](./coraza.md) §9.
22. **Schemathesis runs a two-channel negative generator.** Instead of only mutating the *schema* and asking `hypothesis-jsonschema` to draw from it, it also takes a freshly-drawn *valid* body and surgically violates one keyword in-place — re-validating with `real_validator` to discard mutations that a permissive sibling schema would accept. This catches the false-negative class where pure schema-level mutation produces values another `oneOf` branch happens to validate. See [`schemathesis.md`](./schemathesis.md) §9.
23. **RESTler is a two-language pipeline split on `grammar.py`.** Heavyweight inference runs offline in F# with algebraic data types and multi-indexed producer caches; the engine becomes a thin stateful Python executor whose entire contract is primitive-name match with `restler/engine/primitives.py:34-52`. The producer-consumer dependency inference (`Dependencies.fs:23-52`) is essentially static-analysis-on-a-REST-spec and is what lets RESTler synthesize multi-request chains without recordings. See [`restler-fuzzer.md`](./restler-fuzzer.md) §9.
24. **Trivy-MCP returns a stub, not findings, from scan calls.** The `ScanResponse` is a small `{batch_id, fingerprint, counts, meta, next}` envelope; the LLM is steered to a paginated `findings_list` tool via the `next.tool` hint. Findings live in an LRU keyed by batch UUID with keyset (base64-cursor) pagination. The compact one-character JSON keys plus emoji-legend `meta.finding_schema` make this the reference "envelope-as-prompt" pattern for wrapping a CLI scanner as MCP. See [`trivy-mcp.md`](./trivy-mcp.md) §9.
25. **Vespasian's `crawl.ObservedRequest` is provenance-agnostic.** Burp items, HAR entries, mitmproxy native flows, and live-Chrome CDP events all collapse to the same struct tagged only by `Source`. A fifth ingest path needs only a new `TrafficImporter` — no changes to classifiers or generators. Notable gap: **authentication detection is not implemented** — observed `Authorization` / `Cookie` / `X-API-Key` headers are not surfaced as `securitySchemes` in the emitted OpenAPI spec. See [`vespasian.md`](./vespasian.md) §8-9.
26. **Ruzzy's bridge is a ~250-line C extension that turns Ruby's `RUBY_EVENT_COVERAGE_BRANCH` event hook into a SanCov 8-bit counters table.** Then `Integer` is monkey-patched so pure-Ruby `==`/`<`/`/` operations feed CmpLog via re-exported `__sanitizer_cov_trace_cmp8`. The build-time trick that makes ASan and Ruby coexist in one `LD_PRELOAD`: `ar d` the duplicate `asan_preinit.cc.o` objects out of the sanitizer archive before `--whole-archive`-linking with libFuzzer. Same binary runs against either clang's libFuzzer or LibAFL's `libafl_libfuzzer_runtime` because the integration is purely at the `LLVMFuzzerRunDriver` + SanCov ABI boundary. See [`ruzzy.md`](./ruzzy.md) §9.

### Round 4 findings

27. **OWASP CRS is engine-neutral by design.** The same rules are validated against ModSecurity 2 (Apache), ModSecurity 3 (Nginx), and Coraza (Go) using one FTW YAML corpus with per-engine override files (`tests/regression/coraza-overrides.yaml`, `nginx-overrides.yaml`). Any SecLang-compatible WAF inherits the entire ruleset. Two orthogonal knobs — anomaly threshold and paranoia level — gate the same rules without touching the rule body, via a uniform `skipAfter` template repeated in every file. See [`coreruleset.md`](./coreruleset.md) §9.
28. **Sunshine is a 2,400-line Python file that runs as both CLI and inside Pyodide in the browser** — same parsing code, switched by a custom `__name__ == "__web__"` sentinel set from JavaScript. Permalinks compress the entire SBOM via client-side LZMA into the URL fragment so analyses are shareable with no server. SBOM-spec robustness includes a `bom-ref` reconstruction heuristic (suffix-match `name@version`, fall back to hash-of-JSON) for malformed CycloneDX files. See [`sunshine.md`](./sunshine.md) §9.
29. **Nettacker's YAML modules can run arbitrary Python expressions via `dependent_on_temp_event` + `eval()`.** Steps reference earlier responses with expressions like `dependent_on_temp_event[0]['content'][0]` that are literally `eval()`'d against the saved-event dict (`nettacker/core/lib/base.py:79,100`). Powerful for multi-stage exploit modules; trust boundary worth knowing about. SOCKS proxy is enforced by **monkey-patching `socket.socket`** (`core/socks_proxy.py`) so every subsequent stdlib call (FTP, SMTP, SSH transport, raw TCP) routes through the proxy without per-protocol awareness. See [`nettacker.md`](./nettacker.md) §3, §7.
30. **ElectronSafeUpdater's "two-layer signature" model decouples Apple Developer ID from update-trust root.** App-managed Ed25519 over `sha256(payload)||"-"||version` runs first (`src/signature.js`); macOS Gatekeeper codesign runs second via Squirrel.Mac. A stolen Developer ID alone can't push, and a stolen Ed25519 key alone can't bypass Gatekeeper. Manifest-version-bind bug noted: literal string `"version"` is hard-coded at `src/downloadandverify.js:139`, so only payload signatures are version-bound. See [`electron-safe-updater.md`](./electron-safe-updater.md) §3, §9.
31. **Katana's hybrid engine routes CDP `Fetch.requestPaused` events back through the same HTML/JS parser stack used for static crawling** (`pkg/engine/hybrid/crawl.go:184`). XHR responses, sub-resources and navigations all reach the queue via one code path. Three engines (`standard` / `hybrid` / pure `headless`) share one `common.Shared` (cookie jar, path trie, per-host LRU backoff). Shadow-DOM is pierced via `DOM.getDocument{Pierce:true, Depth:-1}`. See [`katana.md`](./katana.md) §9.
32. **httpx recursively expands scope from probe artefacts** — TLS Subject Alternative Names and CSP `connect-src` domains are fed back into `process` as new targets (`runner/runner.go:1697-1716`), turning one seed into a discovery walk without an external resolver. Favicon hashing reproduces Shodan's exact recipe (base64-with-`\n`-every-76-chars then mmh3 seed 0) so `-favicon` results pivot directly into Shodan/Censys queries (`common/stringz/stringz.go:128`). See [`httpx.md`](./httpx.md) §10.
33. **akto's MCP shield is the broadest OSS hook surface for AI coding agents** — 15 client integrations (Claude CLI / Cursor / Codex / Copilot / Gemini / langchain / vertex-adk / litellm / opencode / neovim / GitHub CLI / etc.) all funnel into a single `/mcp` Struts endpoint with a uniform JSON-RPC validation contract. `MODE=atlas` hijacks `CLAUDE_API_URL` per device for transparent gateway insertion. Multi-tenancy uses `accountId` as the literal Mongo database name (`SingleTypeInfoDao.java:47`). **Hygiene flag:** hardcoded JWT bearer literal at `apps/data-ingestion-service/.../IngestionAction.java:52`. See [`akto.md`](./akto.md) §9.
34. **Cherrybomb attacks a different layer than Schemathesis** — spec-as-document audits (missing `401`/`403` responses, unbounded arrays, `basic` auth, verb/scope mismatches) reachable in CI without ever sending a request. Macro-generated rule registry (`scan/macros.rs:1-117`) ships passive and active rules through the same `impl_*_checks!` template, where the active variant pairs an attack function with an `is_2xx`/`reflected_and_2xx`/`ssrf_and_2xx` response classifier — attack + interpretation declared in one line. See [`cherrybomb.md`](./cherrybomb.md) §8.

## Cross-cutting patterns visible only at the code level

- **Apache 2.0 + Praetorian convention.** Brutus and the other Praetorian repos in the broader stratsession inventory ship a `pkg/sdk/capability.go` adapter (`brutus/pkg/sdk/capability.go:18-46`) so the tool plugs into Praetorian Chariot. The pattern is consistent across `Hadrian`, `Vespasian`, `Julius`, `Brutus`. They're shipping tools as both standalone CLIs and as Chariot capabilities in one repo.
- **Rust + aya + bpf-linker as the modern eBPF stack.** Little Snitch for Linux's `ebpf/` crate is the cleanest reference for "modern eBPF in Rust" — the `common/` crate compiles for both `bpf` target and host via a `user` cargo feature for `aya::Pod` impls. Applicable beyond per-process firewalling.
- **Provider-agnostic IR + Rego policies is the right shape for compliance scanners.** Plumber's `internal/ir/pipeline.go` (struct shared by GitLab and GitHub normalizers) + single OPA evaluation loop is the architecture other CI/CD compliance scanners will converge to.
- **SQLite virtual tables as the SQL frontend for forensics walkers.** mquire (Rust) and the Allama Python codebase both surface SQL over computed data — vtab-as-walker is becoming the canonical pattern. osquery proved it; mquire shows it works for snapshot-based forensics too.
- **CEL is showing up everywhere.** Betterleaks's `celenv/` is three CEL environments (prefilter / filter / validation). Plumber's policies are Rego (different language, same shape — declarative expression layer). Spotter (K8s) is CEL. The Kubernetes admission ecosystem is CEL. Investing in CEL is reusable across multiple tool categories.
- **Two-tier sandboxing where one is the deny-by-default fallback.** Allama's nsjail + AST-allowlist combo is the right shape for any tool that runs untrusted-author code.
- **Closed-vocabulary LLMs + hard validators is the right shape for OSS agentic tooling.** Both Hadrian (`pkg/planner/planner.go:94`) and Buttercup's POVReproducer pattern enforce that any LLM-emitted command/finding gets verified against the tool's own model *before* it ships downstream. The LLM never gets to widen the attack/finding surface beyond what the tool can already do.
- **Wasmtime / WASM as the new sandboxing default.** Buttercup wraps LLM-written Python in wasmtime + WASI (`sandbox/execute_llm_code.py`). Trivy embeds wazero for custom analyzers (`pkg/module/module.go`). Brutus embeds IronRDP-in-WASM driven from Go via wazero. The pattern is converging: when you want isolation without a container or a separate process, WASM is the answer.
- **Single neutral IR + many encoders/decoders.** Trivy's `core.BOM` (one model, CycloneDX/SPDX/in-toto wrappers all encode and decode to/from it) and Plumber's `ir.NormalizedPipeline` (one Rego engine for GitLab and GitHub CI) are the same architectural pattern. Saves enormous downstream complexity.
- **Vendor concentration is real but healthy in OSS security.** Trail of Bits (Buttercup, mquire, Trailmark, Ruzzy), Praetorian (Brutus, Hadrian, Vespasian, Julius), Aqua (Trivy, Trivy MCP), GitHub Security Lab (Taskflow Agent), Boost Security Labs (SmokedMeat, poutine), Doyensec (SafeUpdater), Microsoft Research (RESTler) and ProjectDiscovery (the entire perimeter chain) account for the bulk of the genuinely novel work in the 2025-2026 window. The lone-author pattern (Betterleaks, PatchLeaks) still exists but the production-grade tools are coming from consultancies' post-engagement OSS pipelines.
- **Convention over configuration scales OSS integration counts.** DefectDojo's 221-scanner support is sustained because parsers register by directory convention (drop a file, get a scanner). AFL++'s 12 mutators load by `dlopen`/`dlsym` walk. Seclab Taskflow Agent's YAML loads from any pip-installed Python package via dotted-module paths. Brutus's 24 protocols register via `init()` side effects. The pattern: **make the registry implicit, the contract narrow, and the discovery loop a directory walk** — that's how 200+ integrations stay maintainable.
- **Cross-language ABI boundaries beat in-process polyglot.** Ruzzy bridges Ruby and LibAFL purely at the `LLVMFuzzerRunDriver` + SanCov C ABI boundary. RESTler hands off F# → Python through `grammar.py` as a generated artifact. Brutus drives Rust IronRDP from Go through WASM with 7 host functions. LibAFL ships the libfuzzer drop-in as an `include_bytes!`-embedded prebuilt cdylib. In every case, the language-boundary contract is a tiny ABI surface or a generated file — not direct linkage. Easier to ship, easier to swap one side without breaking the other.
- **Polyglot fan-out is the AI-security shape.** akto's 22 services span Java + Go + Python + TypeScript + Lua because LLM guardrails and traffic analysis pull in ML libraries that live outside the JVM (FastAPI + ONNX + `llm_guard` + SentenceTransformers + Cloudflare Worker + Intel Hyperscan). Allama's nsjail/PydanticAI/Temporal stack and Buttercup's Python/LangGraph/CodeQuery stack show the same shape. The AI-security era doesn't fit one language; expect 5+ language stacks per platform.
- **Wire-format-as-architectural-seam.** RESTler decouples F# compiler from Python engine through `grammar.py`; Vespasian decouples ingest from generate through `capture.json`; akto decouples mirror from runtime through a Kafka JSON topic; Plumber decouples GitLab/GitHub collectors from Rego rules through `ir.NormalizedPipeline`. A serialised intermediate at the architectural seam is what makes each of these systems independently testable and replaceable on one side at a time. Generalisable pattern beyond security tooling.
- **YAML is the security-tooling DSL of choice.** Nettacker, akto, CRS-via-FTW, Plumber, Seclab Taskflow Agent, Hadrian, Schemathesis-via-config, Brutus-via-config, SmokedMeat's LOTP catalog — every category in this brief has at least one tool whose rule/test/agent format is YAML. The actual *capability* differences are entirely in what gets evaluated at YAML parse time vs at runtime — CRS's `chain` is parse-time, Nettacker's `eval()` is runtime, Hadrian's planner-validated steps are runtime+validated. Worth investing in YAML schema design competency.

## Methodology + caveats

- Each report was produced by reading the actual source under the relevant submodule. Where the codebase ships a `CLAUDE.md` / `AGENTS.md` (SmokedMeat, Allama), those files are **contributor instructions** for the upstream project, not source-of-truth for what the code does — they're ignored except as pointers into specific file paths.
- Line numbers refer to the working-tree snapshot at the time of analysis (May 2026 submodule pins). They will drift; treat them as orientation rather than permanent references.
- Some reports flag explicit uncertainties (e.g. Betterleaks notes the validator HTTP path has no per-request retry/backoff and infers the rate-limit handling lives inside CEL). Where claims are ungrounded, the report says so.
- The **SmokedMeat report is architecture-focused** (3-component layout, NATS subjects, BBolt schema, Pantry graph API, dependencies, build choices) rather than enumerating exploitation primitives — those are documented in [`../ci-cd-security.md`](../ci-cd-security.md) from the project README. The framework's per-vector exploitation code is not detailed at the line level here.
