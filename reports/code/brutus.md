# Brutus — code-level analysis

Source: `sources/perimeter/brutus/` (Apache-2.0, Go).

## Architecture overview

The CLI in `cmd/brutus/main.go:83` parses flags and delegates to the public library at `pkg/brutus`. Plugin/analyzer wiring is pure `init()` side-effect: `cmd/brutus/main.go:27-28` blank-imports `_ "internal/analyzers"` and `_ "internal/plugins"`; library consumers pull the same set via `_ "pkg/builtins"` (`pkg/builtins/builtins.go:8-11`). Library entry point `brutus.Brute(*Config) ([]Result, error)` lives at `pkg/brutus/brutus.go:384`, which calls `BruteWithContext` → resolves a `Plugin` via `GetPlugin` → invokes `runWorkers` (`pkg/brutus/brutus.go:350-375`). Second consumer: `pkg/sdk/capability.go:18-46` exposes Brutus as a Praetorian `capability-sdk` capability for Chariot.

## Module / package layout

- `cmd/brutus/` — CLI: flag parsing (`main.go`), nerva-JSON ingestion (`runner.go`), AI routing (`research.go`), output (`output.go`, `usage.go`, `config.go`, `fingerprint.go`).
- `pkg/brutus/` — core library: `Plugin`/`KeyPlugin` interfaces and `Config`/`Result` (`brutus.go`), registries (`registry.go`), worker pool (`workers.go`), embedded wordlists (`defaults.go` + `wordlists/*.txt`, 24 protocols), nerva JSON contract (`nerva.go`), retry/backoff (`backoff.go`), HTTP auth-type detection (`http.go`), LLM glue (`llm.go`).
- `pkg/builtins/builtins.go` — single-file side-effect aggregator.
- `pkg/badkeys/` — embedded SSH bad-keys (`badkeys.go`, `keys/rapid7/*.key`, `keys/vagrant/vagrant.key`).
- `pkg/sdk/` — Chariot capability adapter.
- `internal/plugins/` — 24 protocol dirs plus a `browser` plugin; `init.go` blank-imports them all.
- `internal/analyzers/` — `claude/` (text + vision) and `perplexity/` clients.
- `examples/{basic,browser,llm}` — library-consumption samples.
- `testdata/` — `docker-compose.yml` and `demo/` goat lab.

## Protocol plugin system

Classic init-time registry. Each protocol calls `brutus.Register("name", factory)` from its `init()`; the binary stitches them by blank-importing `internal/plugins`. `pkg/brutus/registry.go:35-44` holds a mutex-guarded `map[string]PluginFactory`; `GetPlugin` (`registry.go:49-60`) returns a fresh instance per call so plugins do not need cross-target locking. Contract is the two-method `Plugin` interface (`Name`, `Test(ctx, target, user, pass, timeout) *Result`) at `pkg/brutus/brutus.go:240-253`; SSH-style protocols may additionally implement `KeyPlugin.TestKey` (`brutus.go:260-265`), detected via a runtime type assertion (`pkg/brutus/workers.go:255`).

Per-protocol implementations are uniformly thin. SSH (`internal/plugins/ssh/ssh.go:38-104`) wraps `golang.org/x/crypto/ssh`, dials via `brutus.DialWithContext`, calls `ssh.NewClientConn`, captures `ServerVersion` into `Banner`, and classifies via a per-protocol indicator list (`ssh.go:30-36`). MySQL (`internal/plugins/mysql/mysql.go:28-99`) opens a `database/sql` connection with `go-sql-driver/mysql`, reads TLS mode from context (`TLSModeFromContext`), pings; classifier at `mysql.go:102-107`. PostgreSQL (`internal/plugins/postgresql/postgresql.go:28-80`) uses `lib/pq` and forces `dbname=postgres` so a missing per-user DB does not masquerade as auth failure. RDP is the outlier — see below.

## Embedded SSH bad keys

PEM files vendored under `pkg/badkeys/keys/rapid7/` (9 files) and `pkg/badkeys/keys/vagrant/` (1) pulled in by `//go:embed keys/rapid7/*.key keys/vagrant/*.key` at `pkg/badkeys/badkeys.go:45-46`. CVE/vendor mapping is a static Go map `keyMetadata` (`badkeys.go:75-220`) keyed by filename — F5 BIG-IP CVE-2012-1493, ExaGrid CVE-2016-1561, Vagrant default, Barracuda CVE-2014-8428, Ceragon CVE-2015-0936, Monroe DASDEC CVE-2013-0137, etc. Recent 2022-2025 entries (Cisco Policy Suite, RUCKUS, Vasion PrinterLogic, AiKaan, Siemens) at `badkeys.go:148-220` are metadata-only stubs — CVE documented but private keys NOT embedded; operators drop them in a dir and load via `LoadSSHCredentialsFromDir` with an optional `badkeys.json` manifest (`badkeys.go:327-436`). `additionalUsernames` (`badkeys.go:224-234`) fans out each key against multiple usernames (e.g., vagrant key → `vagrant,root,ubuntu,centos,ec2-user,admin`). SSH plugin consumes them as paired `Credential{Username, Key}` via `Config.applyDefaults` (`pkg/brutus/brutus.go:288-309`).

## RDP via IronRDP-in-WASM

A 2.0 MB `internal/plugins/rdp/ironrdp.wasm` is embedded at compile time by `embed.go:19-20` (`//go:embed ironrdp.wasm`). The Rust source is in-tree at `internal/plugins/rdp/rust/` — `Cargo.toml` declares `crate-type = ["cdylib"]` on `ironrdp-pdu 0.7`, `ironrdp-connector 0.8`, `ironrdp-session 0.8`, `ironrdp-graphics 0.7`, `ironrdp-input 0.5`; `build.sh` produces the wasm.

Go drives the module through wazero (`internal/plugins/rdp/wasm.go:28-31`). `sync.Once`-guarded `initEngine` (`wasm.go:74-132`) builds a `wazero.Runtime`, instantiates WASI snapshot preview1 (clock/random), and registers seven host functions on the `env` module: `host_tcp_read`, `host_tcp_write`, `host_tls_upgrade`, `host_clock_now_ms`, `host_random_fill`, `host_log`, `host_get_tls_server_pubkey` (lines 88-110), then compiles the embedded bytes. Per-`Test()` isolation: `newInstance` (`wasm.go:144-156`) instantiates a fresh module so each goroutine has its own linear memory; the active `net.Conn` (and post-upgrade `*tls.Conn`) is plumbed to host functions via a typed context key (`wasm.go:42-57`).

Rust exports `wasm_alloc`, `wasm_dealloc`, `connector_new`, `connector_step`, `session_new`, `session_free`. The Go driver at `rdp.go:177-336` marshals an `rdpConfig` JSON (`rdp.go:69-76`) into WASM memory, then loops on `connector_step` switching on five integer states (`stateNeedSend`, `stateNeedRecv`, `stateNeedTLSUpgrade`, `stateConnected`, `stateError` — `wasm.go:33-40`). TLS upgrade for CredSSP is performed by Go (`rdp.go:309-326`); the server's `SubjectPublicKey` BIT STRING is extracted via ASN.1 and handed back to Rust through `host_get_tls_server_pubkey` (`wasm.go:334-364`) — comment notes it matches IronRDP's native `extract_tls_server_public_key`, with sspi-rs needing those raw bytes for the CredSSP `pubKeyAuth` computation.

Sticky-keys T1546.008 detection runs as a second non-NLA connection per target (`rdp.go:386-414`, gated by `shouldRunStickyKeysCheck` at `rdp.go:381-383`). `session.go:195-260` brings up a graphics session at 1024x768, pumps until the login bitmap arrives, sends Left-Shift scancode `0x2A` five times with 50 ms gaps (`session.go:233-243`), waits 1.5 s, captures a second frame. `analyze.go:81-129` computes per-pixel brightness diffs against the baseline (threshold 30 brightness, 2–80% of frame), then `detectChangedRectangle` (`analyze.go:133-183`) checks whether changed pixels fill >40% of their bounding box — terminal windows have dense solid backgrounds. Verdicts: `clean`/`backdoor_likely`/`backdoor_confirmed`/`vulnerable` with a confidence score. If `ANTHROPIC_API_KEY` is set and `--no-vision` not used, `sticky_vision.go:38-138` PNG-encodes the post-keystroke frame and POSTs to `https://api.anthropic.com/v1/messages` with `claude-sonnet-4-5-20250929` (overridable via `BRUTUS_VISION_MODEL`), prompting for one of `BACKDOOR_CONFIRMED`/`STICKY_KEYS_DIALOG`/`NO_CHANGE`/`UNCLEAR`; `BACKDOOR_CONFIRMED` upgrades the heuristic verdict by +0.3 confidence (`analyze.go:230-247`). An identical pipeline in `utilman_vision.go` triggers Win+U (`leftWinScancode 0xE05B` + `uKeyScancode 0x16` from `keyboard.go:30-34`) to detect a backdoored `utilman.exe`.

## Pipeline integration

The nerva contract is `NervaResult` in `pkg/brutus/nerva.go:30-39` (fields: `host`, `ip`, `port`, `protocol`, `tls`, `transport`, `version`, `metadata`). The service-string → plugin-name mapping is a flat `map[string]string` in `MapServiceToProtocol` (`nerva.go:43-87`) — `postgres`/`postgresql` both map to `postgresql`, unknown services return `""` and are skipped. Stdin ingestion is a line-oriented JSONL scanner in `cmd/brutus/runner.go:106-167`: one `NervaResult` per line, mapped to a protocol, then `runSingleTarget` (`runner.go:172-194`) builds a `brutus.Config`. TLS flagged by nerva auto-upgrades local TLS mode via `detectTLS` (`runner.go:298-304`). Valid creds stream out immediately (`runner.go:160-163`) so large naabu/nerva-fed scans do not buffer. A `--fingerprint` mode runs nerva-plugins in-process (`nervaplugins "github.com/praetorian-inc/nerva/pkg/plugins"` at `nerva.go:26`, plus `ParseNervaTarget`/`ServiceToNervaResult` at `nerva.go:92-143`).

## Concurrency / rate limiting

Pool is `golang.org/x/sync/errgroup` with `SetLimit(cfg.Threads)` (`pkg/brutus/workers.go:125-126`); default 10 (`brutus.go:334`). Rate limiting uses `golang.org/x/time/rate.NewLimiter(rate.Limit(cfg.RateLimit), 1)` (`workers.go:130-132`), with optional uniform random `Jitter` after each token grant (`workers.go:193-202`). Connection errors trigger an `attempt < cfg.MaxRetries+1` retry loop with a `backoffController` (500 ms → 30 s, `workers.go:135-138`, in `backoff.go`); the same controller exposes `adaptiveDelay()` that paces the pool when errors spike. `StopOnSuccess` is an `atomic.Bool` "found" flag plus errgroup-scoped context cancel on success (`workers.go:298-301`), with both a pre-launch check (`workers.go:152-154`) and a re-check just before the network call to close the TOCTOU window opened by rate-limit waits (`workers.go:215-222`). `MaxAttempts` per username is mutex-guarded counter map (`workers.go:204-213`). `SprayMode` reorders the credential list so each password is tried across every user first (`workers.go:73-96`). Plugin panics are `recover()`-ed into a synthetic `Result` so one bad target cannot crash the scan (`workers.go:161-178`). No connection pooling — each `plug.Test` opens a fresh TCP connection.

## `--experimental-ai` HTTP admin detection

`--experimental-ai` requires `ANTHROPIC_API_KEY`; `PERPLEXITY_API_KEY` is optional and switches the LLM provider name (`cmd/brutus/main.go:39-50`). Routing in `cmd/brutus/research.go:28-44`: `routeHTTPWithAI` calls `brutus.DetectHTTPAuthType` (`pkg/brutus/http.go:49`); a `WWW-Authenticate: Basic` response keeps the `http`/`https` plugin and uses Claude text analysis on the banner (`workers.go:106-109` → `runWorkersWithLLM` at `workers.go:356-395`), while anything else is rerouted to the `browser` plugin. The browser plugin (`internal/plugins/browser/`) uses `github.com/chromedp/chromedp` (`chrome.go:13,50-127`) with `Flag("headless", !visible)`, `no-sandbox`, `disable-dev-shm-usage`; `NavigateAndScreenshot` captures a PNG via `chromedp.CaptureScreenshot`. Screenshot is sent to Claude Vision through `internal/analyzers/claude/claude.go` (`init` 39-51 registers both `claude` and `claude-vision`; vision request structs at lines 91-120). Perplexity (`internal/analyzers/perplexity/perplexity.go:27-34`) does follow-up "default credentials for product X" lookups and returns paired `Credential` values via the `CredentialAnalyzer` interface (`pkg/brutus/brutus.go:212-215`). Two-stage flow composed in `research.go:76-100` (`researchBrowserCredentials`). `--experimental-ai-verify` additionally turns on before/after screenshot comparison inside the browser plugin (`browser.go:66-68`, `AIVerify` field).

## What's genuinely novel at the code level

Compiling IronRDP's Rust state machines to WASM and driving them from Go through wazero — Go owning the socket, TLS upgrade, and CredSSP public-key extraction via a tiny seven-function host ABI — lets a Go binary speak NLA/CredSSP correctly without rewriting RDP and without CGO. The sticky-keys/utilman backdoor check is a real defensive primitive on top of that: a deterministic pixel-diff + bounding-box-fill heuristic with an optional Claude Vision second opinion that fuses heuristic and LLM verdicts with explicit confidence arithmetic (`analyze.go:230-247`), instead of using the LLM as sole detector. Everything else — registry, worker pool, embedded badkeys, nerva JSONL — is conventional Go, but cleanly split into a reusable library (`pkg/brutus`) plus a Chariot capability adapter (`pkg/sdk`), so the engine is consumable outside the CLI.
